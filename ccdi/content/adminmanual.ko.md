# CCDI Internship Platform 관리자 매뉴얼

이 문서는 **관리자(Admin)** 가 `/admin`에서 **신청서 검토/상태 변경/서류 열람·다운로드** 및 **관리자 계정 관리**를 수행하는 방법을 안내합니다.

---

## 1) 관리자 화면 구성 한눈에 보기

관리자 페이지는 크게 2개의 탭으로 구성됩니다.

- **Applications**: 신청 목록 조회, 상세 보기, 상태 변경, 서류 확인
- **Admins**: 관리자 계정 생성/권한(Role) 변경/활성화/비밀번호 재설정

상단바(Topbar)에는 다음이 있습니다.

- **Home** 버튼: 메인 사이트(`/`)로 이동
- **현재 관리자 이메일 표시**(가능한 경우)
- **Logout** 버튼

> 팁(예시)  
> “서류 확인 → 상태 변경”을 많이 반복한다면, Applications 탭에서 **`SUBMITTED`만 먼저 처리**하고, 다음으로 `UNDER_REVIEW`를 정리하는 흐름이 효율적입니다.

---

## 2) 로그인 / 로그아웃

### 2.1 로그인

1. 브라우저에서 `/admin/login` 접속
2. **Email / Password** 입력 후 `Login` 클릭
3. 성공하면 기본적으로 `/admin/applications`로 이동합니다.

예시

- Email: `admin@example.com`
- Password: `******`

> 로그인 성공 시 서버가 **관리자 쿠키(`ccdi_admin`)** 를 설정하고, 이후 요청은 쿠키 기반으로 인증됩니다.

### 2.2 로그아웃

- 상단바의 `Logout` 클릭 → 서버에 로그아웃 요청 → `/admin/login`으로 이동합니다.

---

## 3) Applications(신청 관리)

### 3.1 신청 목록 보기(`/admin/applications`)

이 화면에서 **신청서 목록을 빠르게 훑고** 상세로 들어갈 수 있습니다.

#### (1) 상태(Status) 필터

오른쪽 드롭다운에서 상태를 선택하면 해당 상태의 신청서만 조회합니다.

- `SUBMITTED` (기본값)
- `UNDER_REVIEW`
- `APPROVED`
- `REJECTED`
- `DRAFT`
- `ALL`

예시

- “접수된 것만 먼저 처리” → `SUBMITTED` 선택
- “검토 중인 것만 다시 보기” → `UNDER_REVIEW` 선택

#### (2) 검색(Search)

검색창에서 아래 항목 기준으로 **화면 내 목록을 필터링**합니다.

- 이름
- 이메일
- 신청서 ID

예시

- `kim` 입력 → 이름/이메일/ID에 `kim`이 포함된 신청서만 화면에 표시

#### (3) 새로고침

`새로고침` 버튼을 누르면 서버에서 목록을 다시 가져옵니다.

#### (4) 상세로 이동

목록에서 항목을 클릭하면 `/admin/applications/[id]` 상세 페이지로 이동합니다.

---

### 3.2 신청 상세 보기(`/admin/applications/[id]`)

상세 페이지에서는 다음을 진행합니다.

- 기본 정보 확인
- 상태 변경 저장
- 서류 미리보기/다운로드

#### (1) 상단 고정 액션 영역

- `목록`: 신청 목록으로 돌아가기
- **현재 상태 배지(StatusBadge)**: 현재 신청 상태 표시
- **상태 드롭다운**: 변경할 상태 선택
- `상태 저장`: 상태 변경을 서버에 반영
- `새로고침`: 상세 정보 재조회

권장 워크플로(예시)

1. 새 신청서 확인 → 상태를 `UNDER_REVIEW`로 변경 → `상태 저장`
2. 서류 확인 후 승인 가능 → `APPROVED`로 변경 → `상태 저장`
3. 보완 필요/부적합 → `REJECTED`로 변경 → `상태 저장`

> 참고  
> 서버 API는 `memo`(메모)도 받을 수 있게 되어 있지만, 현재 화면에서는 메모 입력 UI가 없습니다.

##### 상태 전환 예시 표

| 상황(예시) | 추천 상태 | 이유 |
|---|---|---|
| 제출된 신청서가 막 들어옴 | `UNDER_REVIEW` | “검토 시작” 표시 |
| 필수서류/동의 모두 정상 | `APPROVED` | 승인 처리 |
| 서류 누락/불명확/부적합 | `REJECTED` | 반려 처리 및 추가 안내 가능 |

#### (2) 기본 정보 섹션

화면에 카드 형태로 주요 필드가 표시됩니다.

예: 이름, 이메일, 국적, 연락처, 대학, 전공, 희망 분야/지역 등

#### (3) 서류 섹션(보기/다운로드)

각 문서 행에서:

- `보기` 버튼: **미리보기용 서명 URL**을 받아 모달로 표시
- `다운로드` 버튼: **다운로드용 서명 URL**을 새 탭으로 열어 다운로드

미리보기 지원 형식

- **이미지(JPG/PNG 등)**: 이미지 뷰어로 표시
- **PDF**: 내장 iframe 뷰어로 표시
- 그 외: “미리보기 미지원” 안내 + 새 탭 링크 제공

##### 미리보기 모달(Preview)

- 상단에 파일명 표시
- `다운로드`: 모달에서 바로 다운로드 링크 생성
- `새 탭`: 미리보기 URL을 새 탭으로 열기
- `닫기`: 모달 종료

> ⚠️ 서명 URL은 유효시간이 짧습니다.  
> - 미리보기 URL: 약 **5분**  
> - 다운로드 URL: 약 **2분**  
> 시간이 지나면 만료될 수 있으니, 만료 느낌이 나면 `보기/다운로드`를 다시 눌러 새 URL을 받으세요.

## 4) Admins(관리자 계정 관리)

관리자 계정 관리는 `/admin/admins`에서 수행합니다.

### 4.1 관리자 목록 확인

각 관리자 계정마다 다음 정보가 보입니다.

- 이메일
- role(권한 문자열)
- active(true/false)
- last login
- created

### 4.2 새 관리자 추가

상단 “새 관리자 추가” 폼에서:

- Email
- Password
- Role 선택(ADMIN/SUPERADMIN/REVIEWER/VIEWER)
- Active/Inactive 토글

생성 규칙

- 이메일은 `@` 포함 형식이어야 함
- 비밀번호는 **최소 6자**
- 이메일은 중복 불가

예시

- email: `reviewer1@example.com`
- password: `reviewer123`
- role: `REVIEWER`
- Active: `Active`

> 참고(중요)  
> 역할(Role)은 데이터로 저장되지만, 현재 API 가드에서는 **“역할별 접근 제한”이 구현되어 있지 않습니다.**  
> 즉, 로그인만 되면 같은 기능을 수행할 수 있고, Role은 운영/정책 용도로 먼저 들어가 있는 상태입니다(추후 제한 추가 가능).

### 4.3 활성화/비활성화(계정 잠금)

목록에서 `비활성화/활성화` 버튼으로 토글합니다.

- **Inactive**가 되면 해당 계정은 로그인에서 막힙니다(인증 단계에서 실패).

운영 예시

- 퇴사/업무 종료 → `비활성화`로 계정 차단
- 단기 인턴 관리자 → 기간 끝나면 `비활성화`

### 4.4 비밀번호 재설정

`비번 재설정` 버튼 클릭 → 프롬프트에 새 비밀번호 입력 → 저장

규칙

- 새 비밀번호도 **최소 6자**

예시

- `Temp#2026!` 같은 임시 비번 발급 후, 관리자에게 전달하고 첫 로그인 후 변경 안내

---

## 5) 운영/보안 상 주의사항

- **공용 PC에서는 반드시 Logout**
- 서류 URL은 서명 기반이며 **만료 시간이 짧음**
  - 링크를 복사해 오래 두고 쓰는 방식에는 적합하지 않음
- “비활성화”는 운영 중 가장 빠른 접근 차단 수단

---

## 6) 자주 발생하는 문제 해결

### Q1. 로그인했는데 Applications가 “불러오기 실패”로 떠요

쿠키 기반 인증이 필요합니다. 아래를 확인하세요.

- `/admin/login`에서 다시 로그인
- 브라우저가 **쿠키 차단** 설정이 되어있지 않은지 확인
- 다른 도메인/포트로 프론트-백이 분리된 환경이라면 CORS/쿠키 설정이 맞아야 합니다(개발자 매뉴얼에서 다룸)

### Q2. “보기” 눌렀는데 미리보기/다운로드가 안 돼요

서명 URL이 만료됐거나(특히 다운로드는 2분), 네트워크/스토리지 오류일 수 있습니다.

- 다시 `보기` 또는 `다운로드`를 눌러 새 URL을 발급받기
- `새로고침` 후 재시도

### Q3. 관리자를 만들었는데 그 계정이 로그인 안 돼요

- 계정이 `Inactive`로 생성되었는지 확인
- 비밀번호가 6자 미만이면 생성/변경 자체가 실패합니다(에러 메시지 확인)
