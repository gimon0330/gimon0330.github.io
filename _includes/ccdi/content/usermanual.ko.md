# CCDI Internship Platform 사용자 매뉴얼 (학생 신청자)

이 문서는 **학생(외국인 유학생) 신청자**가 `/apply`에서 신청서를 작성하고, 서류를 업로드한 뒤 **최종 제출**까지 완료하는 방법을 안내합니다.

---

## 1) 한눈에 보는 전체 흐름

1. **홈(`/`) → 학생 신청(`/apply`)** 진입
2. 신청서 작성은 **총 6단계**

   - 1. 개인 정보
   - 2. 학력 정보
   - 3. 실습 희망 정보
   - 4. 제출 서류 업로드
   - 5. 동의 사항
   - 6. 검토 및 제출

3. 최종 제출이 성공하면 **신청서 ID가 표시**되고 접수가 완료됩니다.

---

## 2) 시작하기

### 2.1 홈에서 들어가기

- 홈 화면에서 **학생 신청** 버튼을 눌러 `/apply`로 이동합니다.
- 우측 상단 **언어 선택(LanguageSelect)** 에서 한국어/영어/중국어/일본어로 전환할 수 있습니다.

### 2.2 “임시 세션”이 자동으로 만들어집니다

`/apply`에 처음 들어오면 시스템이 자동으로 “신청서 세션”을 만들고 브라우저에 저장합니다.

- 화면 왼쪽(또는 모바일 상단 카드)에서 **Application ID**가 보입니다.
- 이 세션은 **로그인 없이** 신청서를 이어가게 해 주는 용도입니다.

> ⚠️ 중요한 제한(현재 구현 기준)
>
> - **입력값/업로드 상태가 새로고침(F5) 시 화면에서 복구되지 않습니다.**
>   즉, 서버에는 일부 저장돼도 **화면에 다시 채워지지 않아서** 진행이 막힐 수 있어요.
> - 따라서 **작성 시작 후에는 새로고침/탭 종료/브라우저 종료를 피하는 것을 권장**합니다.

---

## 3) 저장 규칙 (꼭 알아야 하는 포인트)

### 3.1 “다음(Next)”을 눌러야 서버에 저장됩니다

각 단계(1,2,3,5단계)의 입력 내용은 **그 단계에서 `다음` 버튼을 눌렀을 때** 서버로 저장됩니다.

- 단계만 이동(예: 스텝퍼에서 1→6 클릭)하면
  **화면에 적어둔 내용이 서버에 저장되지 않을 수 있습니다.**

✅ 권장 진행 방식

- 각 단계에서 **필수 항목 입력 → `다음` 버튼 클릭 → 다음 단계로 이동**

예시

- Step1에서 이름/국적/생년월일/전화/이메일/여권번호를 다 쓴 뒤
  **반드시 `다음`을 눌러야** Step1 정보가 서버에 반영됩니다.

### 3.2 서류 업로드는 “파일 선택 즉시” 업로드됩니다

Step4의 서류는 `다음`을 누르기 전이라도, **파일을 선택하는 순간 업로드가 진행**됩니다.

---

## 4) 단계별 작성 가이드

## Step 1) 개인 정보

| 항목 | 필수 | 규칙 | 예시 |
| --- | ---: | --- | --- |
| 이름(Full Name) | ✅ | 비어 있으면 불가 | `GIMON KANGHYUN AHN` |
| 국적(Nationality) | ✅ | 목록에서 선택 | `Korea (대한민국)` |
| 생년월일(Date) | ✅ | 날짜 선택 | `2002-03-15` |
| 연락처(Phone) | ✅ | **숫자만 10~11자리**(하이픈 X) | `01012345678` |
| 이메일(Email) | ✅ | 이메일 형식 | `user@example.com` |
| 여권번호(Passport No.) | ✅ | 최소 6자 | `M1234567` |

> 국적 목록이 로딩되지 않는 경우(오류 문구 표시)
>
> - 네트워크/서버 상태 문제일 수 있어요.
> - **아직 아무것도 입력하지 않았다면** 새로고침으로 해결될 때가 있습니다.

---

## Step 2) 학력 정보

대학교/전공은 **한글/영문 둘 다 필수**입니다.

| 항목 | 필수 | 규칙 | 예시 |
| --- | ---: | --- | --- |
| 대학교(국문) | ✅ | 2글자 이상 입력 시 자동검색 | `포항공과대학교` |
| 대학교(영문) | ✅ | 직접 입력/자동채움 | `POSTECH` |
| 전공(국문) | ✅ | 직접 입력 | `컴퓨터공학` |
| 전공(영문) | ✅ | 직접 입력 | `Computer Science` |
| 학번(Student ID) | ⭕ | 선택 | `2024xxxx` |
| 이수학기(Completed Semesters) | ⭕ | 숫자 선택 | `4` |
| GPA | ⭕ | 0~5 범위 | `3.78` |
| TOPIK | ⭕ | TOPIK 1~6 | `TOPIK 4` |
| 비자 유형(Visa Type) | ⭕ | 텍스트 | `D-2` |

대학교 자동검색 팁(예시)

- `포항`처럼 **2글자 이상** 입력하면 추천 목록이 뜹니다.
- 목록에서 선택하면 **국문/영문이 함께 채워집니다.**

---

## Step 3) 실습 희망 정보

| 항목 | 필수 | 규칙 | 예시 |
| --- | ---: | --- | --- |
| 희망 분야 | ✅ | 드롭다운 선택 | `광고, 마케팅` |
| 희망 기간 | ✅ | 3개월/6개월/1년 | `6개월` |
| 희망 시작일 | ✅ | 날짜 선택 | `2026-07-01` |
| 희망 지역 | ✅ | 드롭다운 선택 | `서울` |
| 희망 회사 | ⭕ | 텍스트 | `ABC Studio` |
| 사업자 번호 | ⭕ | 텍스트 | `123-45-67890` |
| 자기소개 | ⭕ | **최대 600자** | (아래 예시 참고) |
| 경험/경력 | ⭕ | 자유 서술 | (자유) |

자기소개 예시(짧게)

> “한국에서 콘텐츠 마케팅 인턴십을 통해 실무 경험을 쌓고 싶습니다. 이전에 동아리 홍보를 담당하며…”

---

## Step 4) 제출 서류 업로드

### 4.1 필수 서류(5개)

- 성적 증명서 (TRANSCRIPT)
- 재학 증명서 (ENROLLMENT)
- 여권 사본 (PASSPORT)
- 비자 사본 (VISA)
- 한국어능력시험 성적표 (TOPIK)

### 4.2 선택 서류(있으면 업로드)

- 포트폴리오 (PORTFOLIO)

### 4.3 업로드 규칙

- 파일 형식: **JPG / PNG / PDF**
- 파일 크기: **10MB 이하**
- 권장사항:

  - 최근 3개월 이내 발급 문서 권장
  - 한글 또는 영문 서류 권장

업로드 예시

- 성적증명서.pdf(2.3MB) → “업로드 완료” 배지가 뜨면 OK

오류가 뜨는 경우

- “jpg/png/pdf 파일만 업로드…” → 파일 형식 확인
- “파일은 10MB 이하…” → 용량 줄이기(스캔 해상도 낮추기 등)
- “업로드 실패: …” → 네트워크/파일 손상/권한 문제 가능(다시 시도)

---

## Step 5) 동의 사항

최종 제출을 위해 **6개 항목 모두 체크가 필요**합니다.

- 개인정보 수집/이용 동의
- 정책 동의
- 고유식별정보 동의
- 민감정보 동의
- 파기 동의
- 제3자 제공 동의

> 하나라도 빠지면 “모두 동의해야 제출 가능” 오류가 표시됩니다.

---

## Step 6) 검토 및 제출

여기서는 제출 전 체크리스트를 확인합니다.

- 필수 항목 입력 완료
- 필수 서류 5개 업로드 완료
- 동의 항목 체크 완료

또한 **입력값 미리보기(JSON)** 를 펼쳐서 확인할 수 있습니다.
(개발/검토용 형태로 보여주며, 제출 전 최종 점검에 도움이 됩니다.)

### 최종 제출

- `제출` 버튼을 누르면 서버에서 아래를 최종 검증합니다:

  - 동의 항목이 모두 체크되었는지
  - 필수 서류 5개가 모두 등록되어 있는지

- 통과하면 신청 상태가 **SUBMITTED(제출됨)** 으로 바뀝니다.

---

## 5) 제출 완료 후(성공 페이지)

제출이 완료되면 성공 화면에서:

- **신청서 ID**가 표시됩니다.
- 이후 절차(요약):

  1. 관리자 서류 확인
  2. 보완 요청 또는 승인 안내(이메일/연락처)
  3. 현장실습 매칭/진행

✅ 권장

- 성공 페이지에 표시된 **신청서 ID를 캡처/메모**해 두세요.
  (문의 시 가장 빠른 식별 정보입니다.)

---

## 6) 자주 발생하는 상황 & 해결법

### Q1. “다 적었는데 다음 단계로 안 넘어가요”

- 필수 입력값 검증에 걸린 경우가 많습니다.

  - 전화번호는 **하이픈 없이 숫자 10~11자리**여야 합니다.
    예: `010-1234-5678` ❌ → `01012345678` ✅
  - 이메일 형식 확인
  - 여권번호 6자 이상 확인

### Q2. Step6에서 제출 버튼이 있는데, Step1~3을 안 거쳐도 제출될까요?

- 화면상 이동은 가능하지만, **서버 저장은 각 단계에서 `다음`을 눌렀을 때** 일어납니다.
- Step1~3을 입력했더라도 `다음`을 안 눌렀다면 서버에 반영되지 않을 수 있어요.
  ✅ 반드시 **각 단계에서 `다음`을 눌러 저장되도록 진행**하세요.

### Q3. 새로고침했더니 입력/업로드 표시가 사라졌어요

- 현재 구현 기준으로는 **화면 복구(재로딩)가 완전하지 않습니다.**
- 가능하면 작성 도중 새로고침을 피하고,
- 이미 새로고침을 했다면:

  - 다시 입력/업로드를 진행해야 할 수 있습니다(특히 Step4 업로드 상태 표시).

---

## 7) 개인정보/보안 안내(사용자 관점)

- 여권번호 같은 민감 정보는 서버에 **암호화 형태로 저장**되도록 설계되어 있습니다.
- 서류는 업로드 후 저장소에 보관되며, 관리자는 열람/다운로드 시 **유효기간이 있는 링크(서명 URL)** 로 접근합니다.
